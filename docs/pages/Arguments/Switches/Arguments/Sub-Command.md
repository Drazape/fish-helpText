# Sub-Command
Different operations from a single command

|      Long     | Short | Exclusive |
| :-----------: | :---: | :-------: |
| `sub-command` |  `c`  | [Positional](Positional "Arguments recognised by their position") |

!!! note "Pairing with *Positional*"
    The program doesn't stop you from using it with [Positional](Positional "Arguments recognised by their position"), but there isn't really a use-case to do so

## Components
- *Header*: Sub-Command
- *Description*: Description of the Sub-Command

## Internal *Description* formatting
Each header matched in the *description*s is automatically italicized
